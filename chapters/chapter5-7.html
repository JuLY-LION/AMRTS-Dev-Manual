Variables (5.7)

Under construction! Come back later!

Variable integers can be used as a trigger when nothing else will fit the situation. They are usually used when an event on one team causes an event on another team, and none of the usual condition types are viable. Integers can be set to any whole number, but you might often find you just need to store a 0 or 1, like a switch. All the following commands are executed from an Action scope.

Creating and updating

Give the variable a name. The <num> is the number value that the variable will start as. The command to create a variable is:

CreateVarInteger("@.[varName]", <num>);

When you're ready to change the value of a variable, the following command is used:

Op("@.[varName]", "=", <num>);

Checking its value

Of course, you're going to want to read the value of your variable. To use it in an conditional if statement, use this:

Condition("If", "@.[varName]", "==", <num>);

Remember, = and == are not the same.
